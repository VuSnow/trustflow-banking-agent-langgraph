TrustFlow Transaction Agent — Main Flow

TransactionAgent uses a ReAct tool-calling loop (max 8 iterations) with 3 tools: text2sql_query (recipient resolution via external text2sql-agent), verify_recipient (DB lookup for name verification), check_fraud_risk (reported_accounts screening). All transactions require OTP. CRITICAL fraud is blocked outright.

FSM State Machine — Confirmation & OTP

WAITING_CONFIRMATION uses an LLM-based classifier (4 classes: CONFIRM / CANCEL / MODIFY / UNCLEAR). WAITING_OTP is fully deterministic — no LLM involved. OTP expires after 300s. Max 3 attempts.
stateDiagram-v2 [*] --> WAITING_CONFIRMATION : Draft frozen WAITING_CONFIRMATION --> WAITING_OTP : CONFIRM (classifier) WAITING_CONFIRMATION --> [*] : CANCEL (classifier) WAITING_CONFIRMATION --> WAITING_CONFIRMATION : MODIFY → ask cancel first WAITING_CONFIRMATION --> WAITING_CONFIRMATION : UNCLEAR → re-ask WAITING_OTP --> OTP_VERIFIED : Valid 6-digit OTP WAITING_OTP --> WAITING_OTP : Invalid OTP (attempts < 3) WAITING_OTP --> BLOCKED : 3 failed attempts WAITING_OTP --> BLOCKED : OTP expired (300s) WAITING_OTP --> [*] : User cancels ("hủy") OTP_VERIFIED --> [*] : Transaction ready BLOCKED --> [*] : Transaction rejected

text2sql_query — Recipient Resolution

The agent uses text2sql_query to search beneficiaries/accounts by name, nickname, or partial info. When multiple candidates are found, the agent asks the user to disambiguate.
